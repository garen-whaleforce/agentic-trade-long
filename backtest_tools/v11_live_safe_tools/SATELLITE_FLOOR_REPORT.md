## 1. 測試背景

### 問題診斷
- **工程稀釋問題**: CAP=12 → CAP=24 時，因 sizing 分母同步放大導致每筆權重被稀釋
- **解決方案**: 解耦 `sizing_positions` 與 `max_concurrent_positions`，實現 Core-Satellite 架構

### 新增功能
1. **sizing_positions**: 獨立的倉位權重計算分母
2. **sort_column**: 同一天訊號排序欄位（使用 `earnings_day_return`）
3. **satellite_floor**: 衛星倉位的最低門檻

## 3. 理論上限分析

### 訊號品質統計 (D7_CORE)
- 平均 30 天報酬: 15.37%
- 正報酬比例: 88.3%
- 最大報酬: 58.17%
- 最小報酬: -37.21%

### 效率差距來源

| 因素 | 影響 |
|------|------|
| 訊號稀疏 | 平均每季 7 筆 (CAP=12 但用不滿) |
| 缺失季度 | 4 個季度無訊號 (2019Q1-Q3, 2021Q1) |
| 曝險不足 | 實際 ~83% vs 理論 200% |
| 負報酬季度 | 6 個季度虧損 (含 2020 COVID 衝擊) |

### 理論 vs 實際
- 理論 CAGR (完美選股 + 2x 槓桿): ~91%
- 實際最佳 CAGR: 29.08%
- 效率比: 32%

3. **動態持倉期**: 根據波動率調整 30 天持倉
4. **Regime 進一步優化**: VIX 門檻精調
